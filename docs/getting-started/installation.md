# Getting Started with AI Transformation

This guide will help you understand how to begin your AI journey with Dracaenium.

## Prerequisites

Before starting an AI transformation initiative, ensure your organization has:

- **Executive Sponsorship**: C-level commitment to AI adoption
- **Clear Business Objectives**: Defined goals and success metrics
- **Data Infrastructure**: Accessible data sources (quality can be improved)
- **Technical Resources**: IT team or willingness to build one
- **Budget Allocation**: Investment capacity for tools, talent, and infrastructure

## Prerequisites

Before installing Dracaenium, ensure you have:

- **Node.js**: Version 18.0 or higher
- **npm** or **yarn**: Latest stable version
- **Git**: For cloning repositories
- **Operating System**: macOS, Linux, or Windows

## Installation Methods

### Method 1: NPM (Recommended)

```bash
# Install globally
npm install -g dracaenium

# Verify installation
dracaenium --version
```

### Method 2: Yarn

```bash
# Install globally
yarn global add dracaenium

# Verify installation
dracaenium --version
```

### Method 3: From Source

```bash
# Clone repository
git clone https://github.com/dracaenium/dracaenium.git

# Navigate to directory
cd dracaenium

# Install dependencies
npm install

# Build from source
npm run build

# Link globally
npm link
```

## Configuration

### Basic Configuration

Create a `dracaenium.config.js` file in your project root:

```javascript
export default {
  // Project settings
  name: 'my-project',
  version: '1.0.0',
  
  // Build configuration
  build: {
    outDir: 'dist',
    sourcemap: true,
  },
  
  // Development server
  server: {
    port: 3000,
    host: 'localhost',
  },
  
  // Plugins
  plugins: [],
};
```

### Environment Variables

Create a `.env` file:

```bash
# API Configuration
DRACAENIUM_API_URL=https://api.dracaenium.com
DRACAENIUM_API_KEY=your_api_key_here

# Database
DATABASE_URL=postgresql://localhost:5432/mydb

# Environment
NODE_ENV=development
```

## Troubleshooting

### Common Issues

#### Issue: Command not found

**Solution:**
```bash
# Ensure npm global bin is in PATH
npm config get prefix

# Add to PATH (macOS/Linux)
export PATH="$PATH:$(npm config get prefix)/bin"
```

#### Issue: Permission denied

**Solution:**
```bash
# Use sudo (not recommended)
sudo npm install -g dracaenium

# Or fix npm permissions
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH
```

#### Issue: Version mismatch

**Solution:**
```bash
# Clear npm cache
npm cache clean --force

# Reinstall
npm uninstall -g dracaenium
npm install -g dracaenium
```
